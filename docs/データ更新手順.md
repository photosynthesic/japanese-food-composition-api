# データ更新手順書

文部科学省の日本食品標準成分表が更新された際の対応手順

## 1. 新しいデータの確認

### 1.1 更新情報の確認
1. [文部科学省のページ](https://www.mext.go.jp/a_menu/syokuhinseibun/index.htm)で新バージョンを確認
2. 更新内容（食品数の変更、新規追加、削除等）をチェック
3. データ形式に変更がないか確認

### 1.2 新しいExcelファイルのダウンロード
```bash
# raw-dataディレクトリに移動
cd raw-data

# 新しいファイルをダウンロード（URLは適宜変更）
curl -L "https://www.mext.go.jp/content/新しいファイル名.xlsx" -o 新しいファイル名.xlsx
```

**重要**: ファイル名は元の形式を保持する（例：`20240428-mxt_kagsei-mext_00001_013.xlsx`）

## 2. 変換スクリプトの更新

### 2.1 ファイルパスの更新
`scripts/convert-excel.js`の171行目付近を更新：

### 2.2 データ構造の変更チェック


**確認項目:**
- シート数（現在19シート）
- 列数（現在62列）
- ヘッダー行の構造
- 特殊値の表記変更

**期待値:**

## 4. データ品質チェック

### 4.1 サンプルデータの確認
```bash
# 最初の3食品を表示
node -e "console.log(JSON.stringify(require('./data/foods.json').slice(0,3), null, 2))"
```

**チェック項目:**
- [ ] 食品名が正しく表示される
- [ ] 数値データが適切に変換されている
- [ ] 特殊値（`"trace"`, `"negligible"`, `null`, `"*"`）が正しく処理されている
- [ ] ヘッダー名が適切（重複や不適切な連結がない）

### 4.2 全データの整合性チェック
```bash
# 必須フィールドの存在確認
node -e "
const foods = require('./data/foods.json');
const required = ['食品群', '食品番号', '食品名'];
const missing = foods.filter(f => required.some(r => !f[r]));
console.log('必須フィールド欠損:', missing.length);
"
```

## 8. トラブル対応

### 8.1 変換エラーが発生した場合
1. `yarn analyze`で構造変更をチェック
2. `scripts/convert-excel.js`のヘッダー処理を調整
3. 新しい特殊値があれば処理ロジックを追加

### 8.2 食品数が大幅に変更された場合
1. 文部科学省の更新情報を再確認
2. 分類変更や統合・分離がないかチェック
3. 必要に応じてカテゴリ処理を調整
