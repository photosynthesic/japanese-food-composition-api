# japanese-food-composition-api Issue 一覧

`shoklog` の `docs/recap.md` §5.1 に基づく、脂肪酸成分表編の取り込み拡張。GitHub Issue は登録済み（#6〜#11）。

## 背景
shoklog（食事ログアプリ）で必要となる **飽和脂肪酸 / n-3系多価不飽和脂肪酸 / n-6系多価不飽和脂肪酸** は、現在同梱している「日本食品標準成分表（八訂）増補2023年」本表には含まれず、別冊「脂肪酸成分表編」にのみ存在する。

本リポジトリを拡張して、本表＋脂肪酸成分表編を **食品番号で join** した形で `foods.json` に同居させる方針。利用側は1か所から取得できる。

## 設計メモ

- **特殊値規約**は既存踏襲：`trace` / `negligible` / `null` / `"*"`。
- **null の意味**：本表には載っているが脂肪酸成分表編に無い食品は、脂肪酸3カラムが `null`。利用側（shoklog）でそれを「未測定」として扱う（recap §9）。
- **収載差分**：脂肪酸成分表編は本表より食品数が少ない。metadata に「脂肪酸データ収載食品数 / 本表食品数」を出して、利用側が欠損率を把握できるようにする。
- **build スクリプト**：既存の `yarn convert` は本表 Excel のみが入力。脂肪酸版を追加するか、`convert` を「本表→脂肪酸 join」の連続ステップに拡張するかは J-2 で決める。
- **コミットメッセージ**：Gitmoji + 英語（shoklog と同じ運用）。
